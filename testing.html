Tests' Description
Unit Testing

"Unit testing is a software testing method by which individual units of source code, sets of one or more computer program modules together with associated control data, usage procedures, and operating procedures, are tested to determine whether they are fit for use"--Wikipedia



Acceptance Test

"Acceptance testing is a test conducted to determine if the requirements of a specification or contract are met.In systems engineering it may involve black-box testing performed on a system "--Wikipedia



Regression testing

"Regression testing is a type of software testing that seeks to uncover new software bugs, or regressions, in existing functional and non-functional areas of a system after changes such as enhancements, patches or configuration changes, have been made to them"--Wikipedia



Conclusion

By comparing different testing tools, we decided to use "Selenium" for our future testing tool for this project. Firstly, it could be used in different platforms. Secondly, it could create a browser-based regression automation test. Therefore in this project, we are planning to use the "Selenium" to test our final website. In addition, we will apply the idea of unit testing on several parts including editing user information, searching by criteria. Apart from using machine testing methods, we are planning to use human testing at the same time to get a thourough testing done for our project


In COMP204P, we did not use testing tools to test for our prototypes, because our prototypes contain limited functionailities, and it would not be very productive to carry out tests on them. Instead, we planned a more detailed testing plan for our final website at the end of term 2.
